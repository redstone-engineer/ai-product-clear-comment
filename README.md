# ai-product-clear-comment

|Поле | Содержание |
| --- | --- |
| Название проекта | GULAG: Gybrid Unified Language Anti-abuse Guard, Гибридная система защиты от деструктивного контента |
| Проблема | Модерация русскоязычных сообществ вручную требует огромных временных и финансовых ресурсов, что приводит к задержкам в реакции на агрессию. Стандартные системы на базе регулярных выражений не справляются с контекстуальной токсичностью, использованием сленга и морфологическими особенностями русского языка. | 
| Целевая аудитория | Администраторы крупных Telegram-каналов, а в будущем разработчики онлайн-игр и владельцы форумов. Сценарий: интеграция модели в конвейер обработки сообщений для мгновенной фильтрации или пометки подозрительного контента. | 
| Пользовательский сценарий | 1. Пользователь отправляет сообщение в чат Телеги → 2. Система анализирует текст через RuBERT и дополнительные слои в реальном времени → 3. В случае превышения порога токсичности сообщение скрывается, а модератор получает уведомление. | 
| Какую задачу решает ИИ | Автоматическая классификация текстовых последовательностей с учетом контекста и семантики, извлечение признаков из глубоких слоев трансформера для выявления скрытой агрессии. | 
| Почему без ИИ хуже | Решения на основе ключевых слов легко обходятся заменой символов или использованием эвфемизмов. ИИ-модель понимает смысл фразы целиком, что позволяет обнаруживать токсичность даже при отсутствии нецензурной лексики. | 
| Тип AI-адаптации | Предсказательная модель: классификация данных на основе обученных паттернов. | 
| Baseline-решение | Использование статических словарей стоп-слов и регулярных выражений (RegEx) для поиска прямых оскорблений. | 
| Целевые метрики | F1​-score, Accuracy >0.92, Latency <100 мс на запрос | 
| MVP на 4-6 недель | 1. Подготовка и очистка набора данных (Kaggle + кастомные данные). 2. Обучение модели RuBERT с реализацией Triple Pooling. 3. Разработка API на RESTAPI для взаимодействия с моделью. 4. Telegram-бот для демонстрации работы. | 
| Риски | Технические: высокая потребность в GPU-памяти для RuBERT. Данные: смещение распределения (сленг меняется быстрее, чем датасет). Безопасность: атаки типа подбора слов для обхода фильтра. |
